## v0.12 status (landed 2026-07-07)

All Tier-1 items below are implemented with tests and verified against the live
cluster (58 ns / 131 workloads / 230 policies) + 67 real-Chromium assertions.
Two sub-parts were consciously scoped down and carried forward:
- **GR-5** — the honesty half shipped (hidden drops counted in `notShownDrops`
  and surfaced in the warnings bar). Rendering the vanished endpoint as a node
  (so the drop draws) is deferred: it needs a new node kind across Go + the JS
  `nodeNS`/`elFor` paths and its own Chromium pass. Tracked here.

  matches no staged allow pair hides in step with the removals when any removal
  is staged, symmetric with the removed deltas). *Precise per-removal*
  attribution of removal-caused additions is deferred — the engine can't cheaply
  say which deletion enabled a given passage (default-deny denials carry no
  provenance), so multi-removal attribution stays coarse. Tracked here.

Measured effects on the live snapshot: GR-1 attributed broad allowances on 299
edges (the traefik removal now projects its full 86-passage footprint); GR-3
split 53 ingress / 51 egress default-deny with the OR badge unchanged; GR-2
changed only 2 dead/phantom refs (both wrongly-scoped before). No live-edge
movement.

## Tier 1 — v0.12 correctness batch  ✅ done (see status above)

### BE-4 · Generated CNP can select a whole namespace (SAFETY)
`pkg/whatif/simple.go:205-235` — `pickSelector` falls back to "full label set"; for a
label-less workload that is `{}` → `endpointSelector: {matchLabels: {}}` selects **every
pod in the namespace**. `peerSelector` (:229-235) degrades the same way to namespace-only.
The manifest is what the user copies to git. Fix: error when no selectable labels exist
("workload ns/name has no stable labels to select on"); reject in GenerateCNPs.

### GR-3 · Default-deny detection misses the sentinel idiom (headline audit corruption)
`pkg/graph/build.go:226-239, :320-328` — detection requires a rule-free body or explicit
`enableDefaultDeny`. Misses (a) the sentinel idiom this cluster itself uses
(`endpointSelector: {}` + one `fromEndpoints: [{matchLabels: {io.cilium.policy/default-deny…}}]`
arm — `len(rule.Ingress) != 0` so never flagged): false "no default-deny" findings, real
half-opens unreported; (b) all-pods netpol with `policyTypes: [Ingress]` + allow rules;
(c) egress-only deny. `Namespace.DefaultDeny` is direction-less — `Audit.HalfOpen`'s
`deny[dstNS]` conflates egress-only deny with ingress deny. Fix: track
`denyIngress`/`denyEgress` per ns (any all-endpoints selector with any section in that
direction, sentinel arms included, or `enableDefaultDeny.<dir>`); keep `defaultDeny` as
the OR for contract compat; use `denyIngress[dstNS]` in ComputeAudit. Add sentinel tests.

### GR-2 · Namespace matchExpressions NotIn/Exists mis-scoped (over-match into excluded ns)
`pkg/graph/build.go:473-484` — only `nsKey + In` is routed to namespace scoping; `NotIn`,
`Exists`, `DoesNotExist` on `io.kubernetes.pod.namespace` fall through to podSel, and pod
labels never contain that key → `NotIn [kube-system]` matches EVERYTHING incl. kube-system;
`Exists` matches nothing (false phantom + DeadRef). Hits this cluster's secret-init CCNPs.
Fix: capture all nsKey operators into a namespace-expression list; evaluate against ns
*name* in match(). Related: `matchLabels[nsKey]` + `In [b,c]` is unioned into nsExact
(:465-477, :486-487) where k8s semantics are AND/intersection — intersect.

### GR-1 · Broad-allowance credit never attributes the enabling policy (footprint root cause)
`pkg/graph/build.go:663-688` — credit pass sets `e.BroadEgress/BroadIngress` booleans but
not `e.Policies`, so `traefik→app` edges list only the app's policy → client footprint
undercounts (9 declared vs 86 engine passages for the traefik CNP). Fix (~15 lines):
`broadEgProv/broadInProv map[string][]string` from the broad edge's policies; append in
credit pass; move sort/compact after crediting. Caveats: (a) consider a parallel field
(`polVia`) to distinguish credited-from-broad from declared-per-pair; (b) exact parity with
engine passages is impossible from edges (pairs with no per-pair edge at all) — document.

### BE-1/BE-2 · Serve robustness: healthz lies, hung collection stalls forever, startup blocks
`cmd/portolan/serve.go:75-141, :162-170, :129, :184` + `main.go:345` — no per-collection
timeout anywhere (rest.Config has none); one blackholed List = no collection ever again,
while /healthz returns 200 forever after first success (prints lastOK, never fails on
staleness → liveness never restarts). First collect() runs before ListenAndServe → probes
get connection-refused → restart-loop exactly while kube-API is degraded. Fix: wrap each
cycle in context.WithTimeout (interval or -collect-timeout); /healthz → 503 when
`time.Since(lastOK) > k*interval` (also surface lastErr); start listener first, initial
collect inside the ticker goroutine.

### GR-5 · DROPPED flows from vanished pods silently hidden
`pkg/graph/flows.go:122-127` — resolve() failure folds flows into undifferentiated NotShown,
*including drops* — drops from crashed/scaled-to-zero pods are exactly the interesting ones.
Fix: for Verdict==DROPPED render the unknown endpoint anyway (whatif.go:128-132 fabricates
bare Pod nodes — same pattern), or split notShownDrops for the honesty banner.

### BE-5/BE-6/BE-7 · whatif API error handling

  → Compute "change set is empty" → HTTP 500 for a user mistake. Error per rule contributing

- engine.go:205,212: `MustAddPolicyEntries`/`MustAddList` panic in the request path on

- serve.go:246: `strings.Contains(err.Error(), "no such policy")` → export
